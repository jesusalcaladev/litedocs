# Command Line Interface (CLI)

LiteDocs comes with a binary package named `create-litedocs` that helps bootstrap the framework.

## Usage

```bash
npx create-litedocs@latest
# or
pnpm dlx create-litedocs@latest
# or
yarn create litedocs
```

## What it does

When you run `create-litedocs`, it performs the following tasks:

1. Asks for a `projectName` (defaults to `my-litedocs-project`).
2. Creates a directory with that name.
3. Bootstraps the necessary files:
   - `package.json` with React, Vite, and LiteDocs dependencies.
   - `vite.config.ts` pre-configured for the LiteDocs Vite plugin.
   - `litedocs.config.js` to define your theme setup.
   - `custom.css` with a full suite of CSS variables for theming.
   - `docs/index.mdx` as your entry point documentation file.
   - Example external components folder (`src/components/`) demonstrating how to use your own React code within MDX.

## Modifying generated code

Once the CLI finishes, you own the code! Feel free to modify the `package.json`, install additional Vite plugins, or configure tailwind, sass, etc. LiteDocs relies upon Vite under the hood, meaning anything you do with Vite, you can do with LiteDocs.
