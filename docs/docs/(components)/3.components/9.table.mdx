# Table

The `Table` component provides a consistent, beautifully styled way to display tabular data in your documentation. It is used throughout LiteDocs to document component properties.

## Basic Usage

You can use the `Table` component by providing `headers` and `data` props.

<Table
  headers={["Prop", "Type", "Default", "Description"]}
  data={[
    ["variant", "primary | secondary", "primary", "The visual style variant."],
    ["size", "sm | md | lg", "md", "The sizing scale of the component."]
  ]}
/>

```tsx
<Table
  headers={["Prop", "Type", "Default", "Description"]}
  data={[
    ["variant", "primary | secondary", "primary", "The visual style variant."],
    ["size", "sm | md | lg", "md", "The sizing scale of the component."]
  ]}
/>
```

## Advanced Usage

You can also use the `Table` component as a wrapper for standard HTML table elements if you need more flexibility.

<Table>
  <thead>
    <tr>
      <th>Feature</th>
      <th>Status</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Standard Markdown</td>
      <td>Supported</td>
    </tr>
    <tr>
      <td>Custom React Components</td>
      <td>Supported</td>
    </tr>
  </tbody>
</Table>

```tsx
<Table>
  <thead>
    <tr>
      <th>Feature</th>
      <th>Status</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Standard Markdown</td>
      <td>Supported</td>
    </tr>
    <tr>
      <td>Custom React Components</td>
      <td>Supported</td>
    </tr>
  </tbody>
</Table>
```

## Sorting

Enable sorting by passing the `sortable` prop. Users can click on headers to sort the data.

<Table
  sortable
  headers={["Name", "Role", "Join Date"]}
  data={[
    ["Alice", "Developer", "2023-01-15"],
    ["Bob", "Designer", "2022-11-20"],
    ["Charlie", "Product Manager", "2023-03-05"],
    ["Diana", "Engineer", "2023-02-10"]
  ]}
/>

```tsx
<Table
  sortable
  headers={["Name", "Role", "Join Date"]}
  data={[
    ["Alice", "Developer", "2023-01-15"],
    ["Bob", "Designer", "2022-11-20"],
    ["Charlie", "Product Manager", "2023-03-05"]
  ]}
/>
```

## Pagination

For large datasets, use the `paginated` prop. You can also customize the number of items per page with `pageSize`.

<Table
  paginated
  pageSize={3}
  headers={["ID", "Title", "Status"]}
  data={[
    ["101", "Initial Setup", "Completed"],
    ["102", "Design Phase", "In Progress"],
    ["103", "Core Logic", "Pending"],
    ["104", "UI Components", "Pending"],
    ["105", "User Testing", "Not Started"],
    ["106", "Deployment", "Not Started"]
  ]}
/>

```tsx
<Table
  paginated
  pageSize={3}
  headers={["ID", "Title", "Status"]}
```

## Props (`TableProps`)

<Field name="headers" type="string[]">
  An array of strings for the table header columns.
</Field>

<Field name="data" type="(string | ReactNode)[][]">
  A 2D array representing the table rows. Each element can be a string or a React node.
</Field>

<Field name="children" type="ReactNode">
  Optional. Standard HTML table content (`thead`, `tbody`, etc) if you want to define the structure manually.
</Field>

<Field name="sortable" type="boolean" defaultValue="false">
  Enables sorting functionality. Users can click on headers to sort the rows.
</Field>

<Field name="paginated" type="boolean" defaultValue="false">
  Enables pagination controls at the bottom of the table.
</Field>

<Field name="pageSize" type="number" defaultValue="10">
  Determines how many rows are displayed per page when `paginated` is `true`.
</Field>

<Field name="className" type="string">
  Additional CSS classes for custom styling of the table container.
</Field>
