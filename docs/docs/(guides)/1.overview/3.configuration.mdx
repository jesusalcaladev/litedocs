# Configuration (`boltdocs.config.js`)

The `boltdocs.config.js` (or `.ts`, `.mjs`) file at the root of your project controls how LiteDocs functions, builds routes, and renders the UI.
We strongly recommend adding a JSDoc type annotation to your JavaScript configuration for auto-complete and type safety:

```js
/**
 * @type {import('boltdocs').BoltdocsConfig}
 */
export default {
  // Your options...
};
```

Below is the comprehensive list of all configuration options according to the `BoltdocsConfig` interface.

## Root Options (`BoltdocsConfig`)

At the top level of your configuration file, you can define the following options:

### `siteUrl`
<Field name="siteUrl" type="string">
The base URL of the site, used primarily for generating absolute paths in your sitemap (if implemented) or canonical URLs.

Example: `"https://boltdocs.dev"`
</Field>

### `docsDir`
<Field name="docsDir" type="string" defaultValue='"docs"'>
The root directory containing your Markdown/MDX documentation files. Change this if you keep your docs in a folder like `"content"`. 

_Note: It is recommended to configure this via the Vite Plugin options in `vite.config.ts`._
</Field>

### `themeConfig`
<Field name="themeConfig" type="BoltdocsThemeConfig" required>
Determines the appearance, navigation, and structure of the theme. _See [Theme Configuration](#theme-configuration) below._
</Field>

### `i18n`
<Field name="i18n" type="BoltdocsI18nConfig">
  Configuration for internationalization.
  - `defaultLocale` (`string`): The fallback locale, e.g., `"en"`.
  - `locales` (`Record<string, string>`): Available locales mapping, e.g., `{ en: "English", es: "EspaûÝol" }`.
</Field>

### `versions`
<Field name="versions" type="BoltdocsVersionsConfig">
  Configuration for documentation versioning.
  - `defaultVersion` (`string`): The fallback version, e.g., `"v2"`.
  - `versions` (`Record<string, string>`): Available versions mapping, e.g., `{ v1: "Version 1.x", v2: "Version 2.x" }`.
</Field>

### `plugins`
<Field name="plugins" type="BoltdocsPlugin[]">
  Array of custom Boltdocs plugins to extend the build process or client-side functionality.
  - `name` (`string`): Unique identifier for the plugin.
  - `enforce` (`"pre" | "post"`): Optional. Run order.
  - `remarkPlugins` / `rehypePlugins` (`any[]`): Extends the MDX parsing pipeline.
  - `vitePlugins` (`VitePlugin[]`): Standard Vite plugins to inject.
  - `components` (`Record<string, string>`): Map of `Name -> Module Path` to globally inject React components into all MDX files.
</Field>

## Theme Configuration

The \`themeConfig\` object controls almost all visual aspects of the default theme.

### `title`
<Field name="title" type="string" defaultValue='"Boltdocs"'>
  The global title of the documentation site, shown in the `<title>` tag and Navbar.
</Field>

### `description`
<Field name="description" type="string" defaultValue='"A Vite documentation framework"'>
  The global site description used for Search Engine Optimization (SEO).
</Field>

### `logo`
<Field name="logo" type="string">
  Relative URL path or endpoint to the site logo displayed in the Navbar.
</Field>

### `navbar`
<Field name="navbar" type="NavbarItem[]" defaultValue='[{ text: "Home", link: "/" }, { text: "Documentation", link: "/docs" }]'>
  Items to display in the top navigation bar. Each item can have the following properties:
  - `text` / `label` (`string`): The display text for the link.
  - `link` / `to` / `href` (`string`): The destination URL. External links (starting with `http`) automatically include an external link icon and open in a new tab.
  - `position` (`"left" | "right"`): Where to align the item. Default is `"left"`.
</Field>

### `sidebar`
<Field name="sidebar" type="Record<string, Array<{ text: string; link: string }>>">
  Custom sidebar groups. Although LiteDocs generates the sidebar automatically based on file-routing, providing this option overrides the automatic generation for specific paths.
</Field>

### `socialLinks`
<Field name="socialLinks" type="Array<{ icon: string; link: string }>">
  Social media icons and links displayed in the Navbar.
  - `icon` (`"discord" | "x" | string`): Identifier for the svg icon.
  - `link` (`string`): Destination URL.
</Field>

### `footer`
<Field name="footer" type="{ text?: string }">
  Simple configuration for the site footer content. Supports HTML strings.
</Field>

### `breadcrumbs`
<Field name="breadcrumbs" type="boolean" defaultValue="true">
  Toggle the rendering of the breadcrumb navigation in the main document layout.
</Field>

### `customCss`
<Field name="customCss" type="string">
  Absolute or relative path to a custom CSS file that overrides the framework's CSS variable values (e.g. `"./custom.css"`).
</Field>

### `editLink`
<Field name="editLink" type="string">
  URL template for an 'Edit this page on GitHub' style link at the bottom of pages. Use `:path` to denote the current file path.

  Example: `"https://github.com/my-org/repo/edit/main/docs/:path"`
</Field>

### `communityHelp`
<Field name="communityHelp" type="string">
  URL to link readers to community support, usually a Discord or GitHub issues link.
</Field>

### `version`
<Field name="version" type="string">
  A static version string explicitly forced into the Navbar (e.g., `"v2.8.9"`). Distinct from the `versions` object used for file-routing.
</Field>

### `githubRepo`
<Field name="githubRepo" type="string">
  Points to your GitHub repo in `owner/repo` format. When provided, the Navbar will automatically fetch and display your GitHub Star count.
</Field>

### `poweredBy`
<Field name="poweredBy" type="boolean" defaultValue="true">
  Toggle the 'Powered by LiteDocs' badge rendered at the bottom of the sidebar.
</Field>

### `codeTheme`
<Field name="codeTheme" type="string | Record<string, string>" defaultValue='"one-dark-pro"'>
  The syntax highlighting theme for code blocks. Supports any Shiki theme (e.g., `"github-dark"`, `"nord"`, `"vsc-dark-plus"`). You can also provide an object for dual-theme support, for example: `{ light: "github-light", dark: "github-dark" }`.
</Field>

### `layoutProps`
<Field name="layoutProps" type="object">
  The most granular control offered over internal UI components without writing a full plugin. You can override or pass inline props to internal elements:
  - `navbar`: Props injected into the `<Navbar>`
  - `sidebar`: Props injected into the `<Sidebar>`
  - `toc`: Props for the Table of Contents right-sidebar
  - `background`: Props for the background component (e.g., gradients)
  - `head`: Data for HTML `<head>` injections
  - `breadcrumbs`: Props for the Breadcrumb component
  - `className`: Global `div` className on the layout container
  - `style`: Inline React styles.
</Field>

## Example Complex Configuration

```ts
import { defineConfig } from 'boltdocs/config';

export default defineConfig({
  siteUrl: "https://boltdocs.dev",
  themeConfig: {
    title: "Awesome Docs",
    description: "The official guide for Awesome App",
    logo: "/logo.png",
    navbar: [
      { text: "Docs", link: "/docs" },
      { label: "Community", href: "https://discord.gg/custom-link", position: "right" },
    ],
    customCss: "./custom.css",
    githubRepo: "boltdocs/boltdocs",
    editLink: "https://github.com/boltdocs/boltdocs/edit/main/docs/:path",
    socialLinks: [
      { icon: "x", link: "https://x.com/boltdocs" },
      { icon: "discord", link: "https://discord.gg/custom-link" }
    ],
    footer: {
      text: "Released under the MIT License."
    }
  },
  i18n: {
    defaultLocale: "en",
    locales: { en: "English", es: "EspaûÝol" }
  }
});
```

# Vite Plugin Options (`vite.config.ts`)

In addition to `boltdocs.config.js`, you can pass specific options directly to the `boltdocs()` Vite plugin inside `vite.config.ts`.

```ts
import { defineConfig } from 'vite';
import { boltdocs } from 'boltdocs';
import react from '@vitejs/plugin-react';

export default defineConfig({
  plugins: [
    react(),
    boltdocs({
      docsDir: "./docs",
      homePage: "./src/HomePage.tsx",
    })
  ]
});
```

### `docsDir`
<Field name="docsDir" type="string" defaultValue='"docs"'>
  The root directory containing your Markdown/MDX documentation files. 
</Field>

### `homePage`
<Field name="homePage" type="string">
  Path to a custom React component (relative to project root) to render at the base route `/`. If not provided, LiteDocs will fall back to using `docs/index.mdx`.
</Field>

### `customCss`
<Field name="customCss" type="string">
  Path to a custom CSS file to override theme variables. Can also be set in `boltdocs.config.js`.
</Field>
