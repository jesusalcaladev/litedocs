# Frontmatter Reference

LiteDocs uses frontmatter to allow you to customize the metadata and sidebar appearance of your pages. Frontmatter is a block of YAML at the very top of your `.md` or `.mdx` files, delimited by `---`.

```markdown
---
title: "My Custom Title"
badge: "New"
---
```

## Available Fields

### `title`
<Field name="title" type="string">
  Overrides the title of the page. By default, LiteDocs infers the title from the filename (stripping numeric prefixes and extensions).
</Field>

### `description`
<Field name="description" type="string">
  Used for SEO meta tags and [Link Previews](/docs/core-concepts/link-previews). This is highly recommended for better search engine indexing and providing context in tooltips.
</Field>

### `permalink`
<Field name="permalink" type="string">
  Overrides the generated URL path for the page. Useful for migrations or preserving SEO ranking when moving files.
</Field>

### `sidebarPosition`
<Field name="sidebarPosition" type="number">
  Controls the sorting order of the page within its section in the sidebar. Lower numbers appear first.
</Field>

> [!TIP]
> You can also use numeric prefixes in filenames (e.g., `1.getting-started.mdx`) to control ordering automatically.

### `badge`
<Field name="badge" type="new | experimental | updated">
  Adds a small decorative badge next to the page link in the sidebar. Useful for highlighting "New", "Stable", or "Deprecated" content.
</Field>

### `icon`
<Field name="icon" type="string">
  Adds an icon next to the page title in the sidebar. Supports Lucide icon names (e.g., `Rocket`, `Book`), raw SVG strings, or image URLs.
</Field>

---

## Section Metadata (in `index.mdx`)

When used in an `index.mdx` file, these fields control the behavior of the entire section/folder.

### `groupTitle`
<Field name="groupTitle" type="string">
  Overrides the name of the section in the sidebar. By default, the folder name is used.
</Field>

### `groupPosition`
<Field name="groupPosition" type="number">
  Controls the sorting order of the entire section relative to other sections in the sidebar.
</Field>

---

## Example Usage

```markdown
---
title: "Advanced Configuration"
description: "Learn how to deep-dive into LiteDocs settings."
badge: New
sidebarPosition: 10
---

# Advanced Configuration

Content goes here...
```
